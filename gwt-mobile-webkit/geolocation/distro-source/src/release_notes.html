Release Notes for @API_VERSION@

Bug fix release

  • Fixed typo in Coordinates.hasAltitudeAccuracy(): altitide -> altitude. See Issue #32.
  • Fixed Position.getTimestamp(): int -> Date. See Issue #29.
  • Added shell script to run sample in compiled mode. See Issue #12.

Release Notes for 0.9.5

Bug fix release

  • Fixed Google Gears issue: An improperly installed Gears plugin touches a blind spot in the library. See Issue #11.
  • Fixed Firefox 3.5 issue: getCurrentPosition() may call callback methods twice (Issue #10).
  • Separated public API from private (implementation) parts, which were never supposed to be public anyway. Cleanup.

Release Notes for 0.9.4

Android was still borked - whenever the Geolocation API was called after onModuleLoad() completed.

  • Added Gears initialisation at two levels (parent + iframe). Tested successfully on Android, Safari + Gears, Firefox 3.5 and iPhone.

Release Notes for 0.9.3

Oversight fix: By error the property provider to determine 'gears' API was borked.

  • Fixed the 'geolocation.api' property provider. Tested successfully on Android (1.5, 1.6 and 2.0.1), Safari + Gears, Firefox 3.5 and iPhone.

Known Issues

  • API is (still) untested on Opera Mobile 9.7.

Release Notes for 0.9.2

Android fix release.

  • Fixed the Geolocation API implementation so it works perfectly with all versions of Android. Due to the iframe design of GWT's default linker the gears_init.js code is applied to the iframe instead of the parent.
  • 'Hardened' the Geolocation methods - added try/catch blocks, to catch potential permission problems (Gears).

Known Issues

  • API is (still) untested on Opera Mobile 9.7.

Release Notes for 0.9.1

Incremental (pre)release of Geolocation API.

  • Added general support for Google Gears' Geolocation API
  • Added support for GWT.getUncaughtExceptionHandler()
  • Fixed commandline scripts

Known Issues

  • API is (still) untested on Opera Mobile 9.7.

Release Notes for 0.9.0

Initial (pre)release of Geolocation API.

  • Works with Opera Mobile 9.7 (according to Opera Mobile 9.7 - features and standards support), iPhone Safari 3.0 and Firefox 3.5

Known Issues

  • API is untested on Opera Mobile 9.7.
